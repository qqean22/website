# Work gallery — folder structure

Images are grouped by **category**, then by **shoot**. Each folder name under a category is one shoot.

## Categories (exact folder names)

Create **exactly** these folders inside `images/work/`:

| Folder name           | Label on site      |
|-----------------------|--------------------|
| `personal-branding`  | Personal branding  |
| `clothing`            | Clothing           |
| `products`           | Products           |
| `events`             | Events             |

## Structure

```
images/work/
├── gallery.json          ← generated by script (do not edit by hand)
├── personal-branding/
│   ├── portrait-session/
│   │   ├── 01.jpg
│   │   └── 02.webp
│   └── headshots-2024/
│       └── ...
├── clothing/
│   └── summer-editorial/
│       └── ...
├── products/
│   └── nike-campaign/
│       └── ...
└── events/
    └── launch-party/
        └── ...
```

- **Category** = one of the four folders above.
- **Shoot** = any subfolder inside a category (e.g. `portrait-session`, `summer-editorial`).
- **Images** = files inside a shoot folder. Supported: `.jpg`, `.jpeg`, `.png`, `.webp`, `.avif`.

Shoot names are shown on the site with spaces and title case (e.g. `portrait-session` → “Portrait Session”).

## Update the gallery

After adding or moving folders/images, run from the **project root**:

```bash
node scripts/update-gallery.js
```

Then refresh the work page (served over HTTP).
